Python<\/strong> is a high-level<\/a>, general-purpose programming language<\/a>. Its design philosophy emphasizes code readability<\/a> with the use of significant indentation via the off-side rule<\/a>.[33]<\/a><\/sup><\/p>\n\n\n\n

Python is dynamically typed<\/a> and garbage-collected<\/a>. It supports multiple programming paradigms<\/a>, including structured<\/a> (particularly procedural<\/a>), object-oriented<\/a> and functional programming<\/a>. It is often described as a “batteries included” language due to its comprehensive standard library<\/a>.[34]<\/a><\/sup>[35]<\/a><\/sup><\/p>\n\n\n\n

Guido van Rossum<\/a> began working on Python in the late 1980s as a successor to the ABC programming language<\/a> and first released it in 1991 as Python 0.9.0.[36]<\/a><\/sup> Python 2.0 was released in 2000. Python 3.0, released in 2008, was a major revision not completely backward-compatible<\/a> with earlier versions. Python 2.7.18, released in 2020, was the last release of Python 2.[37]<\/a><\/sup><\/p>\n\n\n\n

Python consistently ranks as one of the most popular programming languages.[38]<\/a><\/sup>[39]<\/a><\/sup>[40]<\/a><\/sup>[41]<\/a><\/sup><\/p>\n\n\n\n

History<\/h2>\n\n\n\n

<\/a><\/p>\n\n\n\n

<\/a><\/p>\n\n\n\n

The designer of Python, Guido van Rossum<\/a>, at OSCON<\/a> 2006<\/p>\n\n\n\n

Main article: History of Python<\/a><\/p>\n\n\n\n

Python was conceived in the late 1980s[42]<\/a><\/sup> by Guido van Rossum<\/a> at Centrum Wiskunde & Informatica<\/a> (CWI) in the Netherlands<\/a> as a successor to the ABC programming language<\/a>, which was inspired by SETL<\/a>,[43]<\/a><\/sup> capable of exception handling<\/a> and interfacing with the Amoeba<\/a> operating system.[13]<\/a><\/sup> Its implementation began in December 1989.[44]<\/a><\/sup> Van Rossum shouldered sole responsibility for the project, as the lead developer, until 12 July 2018, when he announced his “permanent vacation” from his responsibilities as Python’s “benevolent dictator for life<\/a>“, a title the Python community bestowed upon him to reflect his long-term commitment as the project’s chief decision-maker.[45]<\/a><\/sup> In January 2019, active Python core developers elected a five-member Steering Council to lead the project.[46]<\/a><\/sup>[47]<\/a><\/sup><\/p>\n\n\n\n

Python 2.0 was released on 16 October 2000, with many major new features such as list comprehensions<\/a>, cycle-detecting<\/a> garbage collection, reference counting<\/a>, and Unicode<\/a> support.[48]<\/a><\/sup> Python 3.0, released on 3 December 2008, with many of its major features backported<\/a> to Python 2.6.x[49]<\/a><\/sup> and 2.7.x. Releases of Python 3 include the 2to3<\/code> utility, which automates the translation of Python 2 code to Python 3.[50]<\/a><\/sup><\/p>\n\n\n\n

Python 2.7’s end-of-life<\/a> was initially set for 2015, then postponed to 2020 out of concern that a large body of existing code could not easily be forward-ported to Python 3.[51]<\/a><\/sup>[52]<\/a><\/sup> No further security patches or other improvements will be released for it.[53]<\/a><\/sup>[54]<\/a><\/sup> Currently only 3.7 and later are supported. In 2021, Python 3.9.2 and 3.8.8 were expedited[55]<\/a><\/sup> as all versions of Python (including 2.7[56]<\/a><\/sup>) had security issues leading to possible remote code execution<\/a>[57]<\/a><\/sup> and web cache poisoning<\/a>.[58]<\/a><\/sup><\/p>\n\n\n\n

In 2022, Python 3.10.4 and 3.9.12 were expedited[59]<\/a><\/sup> and 3.8.13, and 3.7.13, because of many security issues.[60]<\/a><\/sup> When Python 3.9.13 was released in May 2022, it was announced that the 3.9 series (joining the older series 3.8 and 3.7) would only receive security fixes in the future.[61]<\/a><\/sup> On September 7, 2022, four new releases were made due to a potential denial-of-service attack<\/a>: 3.10.7, 3.9.14, 3.8.14, and 3.7.14.[62]<\/a><\/sup>[63]<\/a><\/sup><\/p>\n\n\n\n

As of November 2022, Python 3.11 is the stable release. Notable changes from 3.10 include increased program execution speed and improved error reporting.[64]<\/a><\/sup><\/p>\n\n\n\n

Design philosophy and features<\/h2>\n\n\n\n

Python is a multi-paradigm programming language<\/a>. Object-oriented programming<\/a> and structured programming<\/a> are fully supported, and many of their features support functional programming and aspect-oriented programming<\/a> (including metaprogramming<\/a>[65]<\/a><\/sup> and metaobjects<\/a>).[66]<\/a><\/sup> Many other paradigms are supported via extensions, including design by contract<\/a>[67]<\/a><\/sup>[68]<\/a><\/sup> and logic programming<\/a>.[69]<\/a><\/sup><\/p>\n\n\n\n

Python uses dynamic typing<\/a> and a combination of reference counting<\/a> and a cycle-detecting garbage collector for memory management<\/a>.[70]<\/a><\/sup> It uses dynamic name resolution<\/a> (late binding<\/a>), which binds method and variable names during program execution.<\/p>\n\n\n\n

Its design offers some support for functional programming in the Lisp<\/a> tradition. It has filter<\/code>,map<\/code>andreduce<\/code> functions; list comprehensions<\/a>, dictionaries<\/a>, sets, and generator<\/a> expressions.[71]<\/a><\/sup> The standard library has two modules (itertools<\/code> and functools<\/code>) that implement functional tools borrowed from Haskell<\/a> and Standard ML<\/a>.[72]<\/a><\/sup><\/p>\n\n\n\n

Its core philosophy is summarized in the document The Zen of Python<\/a><\/em> (PEP 20<\/em>), which includes aphorisms<\/a> such as:[73]<\/a><\/sup><\/p>\n\n\n\n

    \n
  • Beautiful is better than ugly.<\/li>\n\n\n\n
  • Explicit is better than implicit.<\/li>\n\n\n\n
  • Simple is better than complex.<\/li>\n\n\n\n
  • Complex is better than complicated.<\/li>\n\n\n\n
  • Readability counts.<\/li>\n<\/ul>\n\n\n\n

    Rather than building all of its functionality into its core, Python was designed to be highly extensible<\/a> via modules. This compact modularity has made it particularly popular as a means of adding programmable interfaces to existing applications. Van Rossum’s vision of a small core language with a large standard library and easily extensible interpreter stemmed from his frustrations with ABC<\/a>, which espoused the opposite approach.[42]<\/a><\/sup><\/p>\n\n\n\n

    Python strives for a simpler, less-cluttered syntax and grammar while giving developers a choice in their coding methodology. In contrast to Perl<\/a>‘s “there is more than one way to do it<\/a>” motto, Python embraces a “there should be one\u2014and preferably only one\u2014obvious way to do it” philosophy.[73]<\/a><\/sup> Alex Martelli<\/a>, a Fellow<\/a> at the Python Software Foundation<\/a> and Python book author, wrote: “To describe something as ‘clever’ is not<\/em> considered a compliment in the Python culture.”[74]<\/a><\/sup><\/p>\n\n\n\n

    Python’s developers strive to avoid premature optimization<\/a> and reject patches to non-critical parts of the CPython<\/a> reference implementation that would offer marginal increases in speed at the cost of clarity.[75]<\/a><\/sup> When speed is important, a Python programmer can move time-critical functions to extension modules written in languages such as C; or use PyPy<\/a>, a just-in-time compiler<\/a>. Cython<\/a> is also available, which translates a Python script into C and makes direct C-level API calls into the Python interpreter.<\/p>\n\n\n\n

    Python’s developers aim for it to be fun to use. This is reflected in its name\u2014a tribute to the British comedy group Monty Python<\/a>[76]<\/a><\/sup>\u2014and in occasionally playful approaches to tutorials and reference materials, such as the use of the terms “spam” and “eggs” (a reference to a Monty Python sketch<\/a>) in examples, instead of the often-used “foo” and “bar”<\/a>.[77]<\/a><\/sup>[78]<\/a><\/sup><\/p>\n\n\n\n

    A common neologism<\/a> in the Python community is pythonic<\/em>, which has a wide range of meanings related to program style. “Pythonic” code may use Python idioms well, be natural or show fluency in the language, or conform with Python’s minimalist philosophy and emphasis on readability.
